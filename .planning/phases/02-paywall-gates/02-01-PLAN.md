   def _anon_rate_limit_key(request: Request) -> str:
       """Return real IP for anonymous requests; fixed high-ceiling bucket for authenticated.

       Authenticated users send an Authorization header via authFetch. They share a
       single bucket keyed "authenticated:bypass" which will never hit the 3/day
       ceiling. Anonymous users get a per-IP bucket like "anon:203.0.113.42".
       """
       if request.headers.get("authorization"):
           return "authenticated:bypass"
       return f"anon:{get_client_ip(request)}"
   ```

   @router.post("/analyze")

   @limiter.limit("3/day", key_func=_anon_rate_limit_key)

       request: Request,
       body: AnalyzeRequest,
       db: Session = Depends(get_db),
       current_user: User | None = Depends(get_current_user_optional),
   ):
   ```

The existing 5/minute global limit stays unchanged and applies to all callers. The new 3/day limit uses the custom key function so only anonymous callers (no Authorization header) are tracked per-IP.

PITFALL GUARD (from RESEARCH.md Pitfall 2): The `_anon_rate_limit_key` function returns `"authenticated:bypass"` for authenticated users -- this means all authenticated users share one bucket. <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> /analyze API | Untrusted input: URL to scan, absence of auth header signals anonymous caller |
| IP header chain | CF-Connecting-IP -> X-Forwarded-For -> client.host -- spoofable without Cloudflare |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Denial of Service | /analyze endpoint | mitigate | 3/day IP rate limit for anonymous callers via `_anon_rate_limit_key`; existing 5/minute global limit preserved. slowapi handles atomic increment and 429 response. |
| T-02-02 | Tampering | _anon_rate_limit_key | mitigate | Key function checks `Authorization` header presence, not validity. An attacker could send a fake Authorization header to bypass the daily limit. However: (1) the backend's `get_current_user_optional` will resolve `current_user=None` for invalid tokens, meaning the scan still runs as anonymous but bypasses the daily limit; (2) the 5/minute global limit still applies; (3) each scan costs ~$0.30-0.57 in Playwright infra, so sustained abuse is self-limiting. Accept residual risk -- the per-minute limit caps throughput. |
| T-02-03 | Spoofing | IP extraction | accept | `get_client_ip()` uses CF-Connecting-IP which Cloudflare controls. In non-Cloudflare environments (dev), falls back to X-Forwarded-For then client.host. Spoofing CF-Connecting-IP requires bypassing Cloudflare -- accepted risk for this deployment model. |
| T-02-04 | Elevation of Privilege | getDimensionAccess | accept | Function now returns "unlocked" for all tiers. This is intentional per D-07 -- the gate moves to the call site (isAnonymous check). No server-side data exposure since anonymous scans already return full scan JSON from the backend. |
</threat_model>
