## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding within badges |
| sm | 8px | Tight element spacing, scan count highlight padding (`px-3 py-2`) |
| md | 16px | Default element spacing, card internal padding base |
| lg | 24px | Card padding (`p-6`), feature list spacing |
| xl | 32px | Card padding large (`p-8`), hero margin bottom |
| 2xl | 48px | Section vertical padding (`pb-12`, `pt-16`) |
| 3xl | 64px | Page hero top (`pt-16 sm:pt-24`) |

Exceptions:
- Touch targets for CTA buttons: minimum 44px height (Button `size="md"` is `py-3` = 48px total, satisfies this)
- "Coming Soon" badge absolute offset: `-top-3` (12px) — intentional visual overlap, not a spacing unit

## Typography

Source: detected from `pricing/page.tsx` class names + `globals.css` font declarations.

| Role | Size | Weight | Line Height | Font Family | Tailwind Class |
|------|------|--------|-------------|-------------|----------------|
| Body | 16px | 400 (regular) | 1.5 | Inter | `text-base` |
| Label/small | 14px | 400 (regular) | 1.5 | Inter | `text-sm` |
| Subheading | 18px | 800 (extrabold) | 1.4 | Manrope | `font-display text-lg font-extrabold` |
| Display/H1 | 48px (desktop) / 36px (mobile) | 800 (extrabold) | 1.1 | Manrope | `font-display text-4xl sm:text-5xl font-extrabold leading-[1.1]` |

Rules:
- All headings use `font-display` (Manrope). Body and labels use default (Inter).
- Exactly 2 weights in use: 400 (regular) for body/label, 800 (extrabold) for all headings.
- `tracking-tight` applied to H1 only.
- `text-[var(--on-surface)]` for primary text; `text-[var(--on-surface-variant)]` for secondary/muted text.

## Color

Source: `globals.css` token definitions. Roles confirmed from `pricing/page.tsx` usage.

| Role | Token | Hex | Usage |

**Pro card (teaser) color treatment:**
- Border: `--outline-variant` (#c6c5d9) — not the brand border
- Background: `--surface-container-lowest` (#ffffff) with `opacity-70`
- Icon background: `--surface-container-high` (#d4d4d8)
- Icon color: `--on-surface-variant` (#454556)
- Badge background: `--surface-container-high` (#d4d4d8)
- Badge text: `--on-surface-variant` (#454556)
- CTA button: `variant="secondary"` (bordered, not filled)

**Confirmation state (waitlist confirmed — authenticated user):**
- Inline text using `--success` (#16a34a) + `--success-light` (#f0fdf4) background pill

## Layout Contract

### Page structure (top to bottom)
1. Hero section — centered text, `pt-16 sm:pt-24 pb-12 sm:pb-16`
2. Card grid section — `pb-16 sm:pb-24`
3. FAQ/trust section — `py-16 sm:py-20 bg-[var(--surface-container-low)]`
4. Footer

### Card grid
- Container: `max-w-3xl mx-auto` (narrowed from current `max-w-7xl` — 2 cards need less width)
- Grid: `grid grid-cols-1 sm:grid-cols-2 gap-4 sm:gap-5 lg:gap-6`
- Stacked (mobile): Free card above, Pro card below
- Side-by-side (sm+): Free card left, Pro card right (source: D-04)

### Card anatomy (both cards share this structure)
1. Optional badge (absolute, `-top-3 left-1/2 -translate-x-1/2`) — Free card: none; Pro card: "Coming Soon"
2. Icon + plan name row (`gap-3 mb-4`)
3. Plan name display (no price row — free plan has no price, Pro is waitlist)
4. Description (`text-sm`, `mb-6`)
5. Scan count highlight pill (`px-3 py-2 rounded-xl bg-[var(--surface-container-low)] mb-6`)
6. Feature list with checkmarks (`space-y-3 mb-8 flex-1`)
7. CTA button (full width, `shape="pill"`)

### Free card — active state
- Border: `border-[var(--brand)]`
- Ring: `ring-2 ring-[var(--brand)]/20`
- Shadow: `shadow-[var(--shadow-brand-md)]`
- No opacity reduction

### Pro card — teaser/muted state
- Border: `border-[var(--outline-variant)]`
- No ring, no brand shadow
- `opacity-70` on card container
- "Coming Soon" badge: `bg-[var(--surface-container-high)] text-[var(--on-surface-variant)]`

## Interaction Contract

### "Get Started" button (Free card)
- Renders as `<Link href="/">` via `Button asChild`
- No state, no modal — pure navigation
- Source: D-03

### "Join Waitlist" button (Pro card)
- **Unauthenticated user:** Opens `AuthModal` with `callbackUrl="/pricing"`
- **Authenticated user (Phase 1):** Sets local React state `waitlistConfirmed = true`, button is replaced by inline confirmation text (no backend call — Phase 3 wires this)
- **Confirmation UI:** Inline within card, replaces button. Text: "You're on the list! We'll let you know when Pro launches." Styled with `--success` text color.
- Source: D-07 + RESEARCH.md Pitfall 5

### Hover / Focus states
- Both CTA buttons: `hover:scale-[1.02]` (inherited from Button component)
- Cards: no hover lift on this page (static pricing, not interactive cards)
- Focus ring: `polish-focus-ring` class (2px outline at `--brand`, offset 2px) — inherited from Button

### Responsive behaviour
- Hero H1: `text-4xl sm:text-5xl md:text-6xl` (scales up)
- Card grid: single column on mobile, 2-col on sm+
- Card padding: `p-6 sm:p-8`
- CTA buttons: `w-full` (full card width)
